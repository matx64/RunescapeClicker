# Phase 0 Acceptance Matrix

This matrix turns the frozen Rust baseline in [PHASE-0-REFERENCE.md](C:/Users/mathe/Documents/dev/RunescapeClicker/PHASE-0-REFERENCE.md) into explicit porting targets for the future Windows-native implementation in [NATIVE-PLAN.md](C:/Users/mathe/Documents/dev/RunescapeClicker/NATIVE-PLAN.md).

## Matrix Columns

| Column | Meaning |
| --- | --- |
| Rust source | The current Rust test or workflow source that defines the behavior. |
| Current behavior | What the legacy app does today. |
| Preserve / drop / redesign | Whether v1 keeps the behavior, intentionally drops it, or redesigns the interaction while preserving intent. |
| Future C# test or WinUI replacement | The expected C# contract test, view-model test, or WinUI replacement to build later. |
| Notes / gaps | Clarifications, migration constraints, or known behavior differences to account for. |
